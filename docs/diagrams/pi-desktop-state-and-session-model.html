Terminology lock

pi-desktop state and Pi session model

This page uses one stable set of names. The goal is to separate pi-desktop UI data from Pi agent configuration and Pi session files.

Default alignment Web dev and Electron dev use the same pi-desktop workspace store path by default.
Real Pi runtime Both use the real Pi SDK session runtime unless smoke mode is enabled.
Important boundary Active running sessions are process memory. Persisted history is Pi session files.

The three persisted things

These names are the glossary. Avoid replacing them with “store” or “sessions” without the qualifier.

pi-desktop workspace store

Projects and chat metadata

The app’s own UI state. It is not the Pi agent conversation history.

  • Projects
  • Chat metadata
  • Selected project and selected chat
  • File: project-store.json
Pi agent config directory

Auth, models, settings, resources

The Pi SDK reads this directory to know how to run real agent sessions.

  • Default: ~/.pi/agent
  • auth.json
  • models.json
  • settings.json
Pi session files

Persisted conversation history

JSONL files created by the Pi SDK session manager for a workspace cwd.

  • Default parent: ~/.pi/agent/sessions
  • Per-cwd encoded subdirectories
  • Files contain Pi conversation/session entries

Runtime memory versus persisted files

This is the part that can sound like “sessions” means two things.

Active Pi session runtime

A currently running SDK session object in one backend process. Electron and web preview do not share this memory if they run separate backends.

Pi session files

Persisted JSONL history under ~/.pi/agent/sessions. Both dev modes can read and write the same files when configured to the same Pi agent paths.

Implementation invariant

The implementation should keep these true by default.

Default dev behavior:
  Electron dev and web dev use the same pi-desktop workspace store.
  Electron dev and web dev use the same Pi agent config directory.
  Electron dev and web dev use the same Pi session files.

Override behavior:
  PI_DESKTOP_USER_DATA_DIR controls only the pi-desktop workspace store.
  PI_CODING_AGENT_DIR controls the Pi agent config directory.
  PI_CODING_AGENT_SESSION_DIR controls the Pi session files directory.

Terminology rule:
  Never say "store" alone.
  Never say "sessions" alone.
  Say "pi-desktop workspace store", "Pi agent config directory", or "Pi session files".
